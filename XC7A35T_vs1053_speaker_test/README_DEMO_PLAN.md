# FIR 插值滤波器赛题演示方案清单

## 1. 演示目标

本项目面向“高阶数字插值滤波器设计与验证”赛题，目标是在 FPGA 平台上完成 24bit 有符号音频采样信号的插值滤波处理，并通过板级实验直观展示 4x、8x、128x 三档输出结果。

系统支持两类输入采样率家族：

* 44.1kHz 家族：

  * 4x 输出：176.4kHz
  * 8x 输出：352.8kHz
  * 128x 输出：5.6448MHz

* 48kHz 家族：

  * 4x 输出：192kHz
  * 8x 输出：384kHz
  * 128x 输出：6.144MHz

## 2. 总体展示结构

本项目采用“两条展示线路”：

### 2.1 主验证线路：AD9708 + 示波器

用途：证明 FPGA 内部 FIR 插值链路的输出采样率、输出波形和模式切换正确。

数据路径：

```text
正弦 ROM / 音频 PCM ROM
        ↓
24bit signed 输入采样
        ↓
FPGA FIR 插值链
        ↓
4x / 8x / 128x 插值输出
        ↓
AD9708 高速 DAC
        ↓
示波器观察输出时钟和模拟波形
```

该线路是赛题答辩的主证据。

### 2.2 辅助听感线路：VS1053 + HT6872 + 板载小喇叭

用途：让评委直观听到“插值前/插值后”音频片段的差异，作为辅助展示。

数据路径：

```text
插值前音频片段 / 插值后音频片段
        ↓
FPGA SPI 发送音频数据
        ↓
VS1053 解码
        ↓
HT6872 功放
        ↓
板载 4Ω/2W 小喇叭
```

注意：该线路用于听感辅助，不作为 4x/8x/128x 采样率的主验证证据。主验证仍然使用 AD9708 + 示波器。

## 3. 演示模式设计

### 模式 A：正弦输入示波器验证

目的：验证插值链路和两类采样率家族是否正常。

输入：

```text
内部 64 点正弦 ROM
24bit signed
ROM_ADDR_STEP 可调
```

输出：

```text
AD9708 模拟输出
示波器观察
```

拨码功能：

```text
sw2 = 0：44.1kHz 家族
sw2 = 1：48kHz 家族

sw1 sw0 = 00：4x 输出
sw1 sw0 = 01：8x 输出
sw1 sw0 = 10：128x 输出
sw1 sw0 = 11：128x 输出
```

预期测量结果：

```text
sw2 = 0：
00：约 176.4kHz
01：约 352.8kHz
10/11：约 5.6448MHz

sw2 = 1：
00：约 192kHz
01：约 384kHz
10/11：约 6.144MHz
```

### 模式 B：音频 PCM 输入示波器验证

目的：证明输入不仅可以是正弦，也可以是真实音频采样。

输入：

```text
短音频片段转换得到的 24bit signed PCM ROM
采样率 44.1kHz 或 48kHz
```

输出：

```text
FPGA FIR 插值链输出
AD9708 模拟输出
示波器观察音频波形包络和输出采样时钟
```

说明：

该模式仍然走 AD9708 + 示波器，是主验证线路的一部分。

### 模式 C：板载小喇叭 A/B 听感辅助展示

目的：辅助展示插值滤波对音频重构质量的影响。

输入：

```text
音频 A：插值前 / 低质量重构版本
音频 B：插值后 / FIR 优化版本
```

输出：

```text
VS1053 解码
HT6872 功放
板载小喇叭播放
```

控制方式：

```text
按键或拨码选择：
A：播放插值前音频
B：播放插值后音频
```

说明：

该模式不直接播放 6.144MHz 的 128x FIR 输出，而是播放可被 VS1053 接收的音频流。它用于听感辅助展示，不替代 AD9708 + 示波器验证。

## 5. 开发步骤

### Step 1：冻结当前 AD9708 示波器验证版本

目标：

```text
保留当前已经编译成功、sw2 可切换两个频率家族的工程。
```

操作：

```text
1. 备份当前 Vivado 工程。
2. 记录当前 bitstream 文件。
3. 记录示波器测得的 4x、8x、128x 频率。
4. 保存示波器截图。
```

### Step 2：整理当前演示表格

目标：

```text
形成一张答辩时可直接展示的模式表。
```

内容：

```text
sw2
sw1 sw0
频率家族
插值倍率
理论输出频率
实测输出频率
误差
```

### Step 3：新增音频 PCM ROM 输入

目标：

```text
把内部正弦 ROM 替换为短音频 PCM ROM。
```

操作：

```text
1. 准备 1~3 秒短 wav 音频。
2. 转成 24bit signed PCM 数据。
3. 生成 .mem 文件。
4. FPGA 按 44.1kHz/48kHz 读出音频采样。
5. 送入现有 FIR 插值链。
6. AD9708 输出到示波器。
```

### Step 4：完成音频 PCM 示波器验证

目标：

```text
证明真实音频采样也能经过 FIR 插值链输出。
```

观察内容：

```text
1. 输出采样时钟是否仍为 4x、8x、128x。
2. 输出波形是否随音频数据变化。
3. 切换 44.1kHz/48kHz 家族是否正常。
```

### Step 5：开发 VS1053 最小播放工程

目标：

```text
让板载小喇叭先能发声。
```

操作：

```text
1. 确认 VS1053 的 FPGA 管脚。
2. 写 SPI 写寄存器逻辑。
3. 复位 VS1053。
4. 设置 VS1053 音量。
5. 设置 VS1053 GPIO4 = 1，打开 HT6872 功放。
6. 根据 DREQ 向 VS1053 发送音频数据。
```

### Step 6：实现 A/B 听感辅助展示

目标：

```text
用板载小喇叭播放插值前/插值后音频片段。
```

操作：

```text
1. 准备音频 A：插值前版本。
2. 准备音频 B：插值后版本。
3. FPGA 根据按键或拨码选择播放 A 或 B。
4. 答辩时说明该支路是听感辅助展示。
```

## 6. 答辩表述建议

可以这样介绍系统：

```text
本设计采用两条验证支路。

第一条为主验证支路：FPGA 内部产生 24bit signed 输入采样，
经多级 FIR 插值链得到 4x、8x 和 128x 输出，
再通过 AD9708 高速 DAC 输出到示波器。
该支路用于验证输出采样率、波形连续性以及两类音频采样率家族的切换。

第二条为辅助听感支路：为了增强展示直观性，
系统额外使用板载 VS1053 与 HT6872 功放播放插值前后音频片段，
用于辅助说明插值滤波对音频重构质量的改善。
核心指标验证仍以 AD9708 + 示波器结果为准。
```

## 7. 现场演示顺序

### 演示 1：正弦 ROM 输入版

目的：展示插值链路的基本正确性。

操作：

1. 下载 `sine_rom_dual_family_ad9708_ok.bit`。
2. 将示波器 CH1 接 AD9708 模拟输出。
3. 将示波器 CH2 接 `dac_clk`。
4. 切换 `sw2`，验证 44.1kHz / 48kHz 两个采样率家族。
5. 切换 `sw1 sw0`，验证 4x / 8x / 128x 三档输出。

预期结果：

| sw2 | sw1 sw0 | 输出倍率 | 理论 dac_clk |
|---|---|---|---|
| 0 | 00 | 4x | 176.4kHz |
| 0 | 01 | 8x | 352.8kHz |
| 0 | 10/11 | 128x | 5.6448MHz |
| 1 | 00 | 4x | 192kHz |
| 1 | 01 | 8x | 384kHz |
| 1 | 10/11 | 128x | 6.144MHz |

### 演示 2：音频 PCM 输入版

目的：展示输入信号不仅可以是理想正弦，也可以是真实音频 PCM 采样。

操作：

1. 下载 `audio_pcm_dual_family_ad9708_ok.bit`。
2. 示波器 CH1 接 AD9708 模拟输出。
3. 示波器 CH2 接 `dac_clk`。
4. 观察 CH1 上复杂音频波形。
5. 切换 `sw1 sw0`，确认 dac_clk 仍为 4x / 8x / 128x。
6. 切换 `sw2`，确认 44.1kHz / 48kHz 家族仍然可切换。

说明：

音频 PCM 输入版使用 1024 点 24bit signed PCM ROM。该版本用于证明真实音频采样数据可以进入 FPGA FIR 插值链，并通过 AD9708 输出到示波器。
